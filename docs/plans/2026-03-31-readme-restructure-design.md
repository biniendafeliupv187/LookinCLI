# LookinCLI MCP README 重组设计

日期：2026-03-31

## 目标

重组 `LookinCLI/mcp/README.md`，让它准确反映当前未推送代码已经具备的能力，并让首次使用者能够按 README 直接完成安装、连接、定位节点、查看详情、读取运行时信息和执行低风险调试操作。

## 范围

只覆盖当前代码已经支持的内容：

- `lookin` CLI
- `lookin-mcp` MCP server
- 当前已实现的命令能力
- 已经验证过的标识符规则与常见边界

不包含：

- 未来规划
- 内部实现细节
- 测试过程或 RavenSpec 过程说明
- 尚未进入代码的能力设想

## README 结构

1. 项目简介
2. 环境要求与安装
3. 快速开始
4. 核心命令总览
5. 常见使用场景
6. 标识符规则
7. 推荐调用顺序
8. 常见限制与排障
9. MCP 与 skill 接入
10. 开发者说明

## 设计原则

- 保留现有安装和接入路径，避免无关改写
- 用“按任务分组”的方式组织命令，而不是按源码模块罗列
- 明确区分 `oid`、`layerOid`、`viewMemoryAddress`、`recognizerOid`
- 只写当前代码里真实存在的行为和限制
- 示例命令优先选择低风险、可直接复制运行的形式

## 预期结果

- 用户能快速理解 CLI 与 MCP 的关系
- 用户能根据 README 正确选择命令和参数
- 用户能避免最常见的标识符误用
- 用户能在遇到 discovery 抖动或图片抓取失败时做出正确判断
